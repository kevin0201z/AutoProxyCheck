# 自动代理守护需求

## 1. 目标

本项目用于在 Windows 系统中守护机器级网络代理。

默认场景如下：

1. 系统平时通过代理上网。
2. 当代理服务器不可用时，系统自动切换到无代理直连模式。
3. 当代理服务器恢复可用时，系统自动重新启用系统代理。
4. 后台检测在开机未登录阶段也能运行。
5. 用户登录后显示系统托盘，用于展示当前代理状态并提供退出入口。

## 2. 范围

本项目负责维护机器级代理的启用状态，并在部署时写入机器级代理地址、端口和绕过列表。

本项目不负责修改以下内容：

- PAC 脚本
- 其他浏览器私有代理参数

## 3. 功能需求

### 3.1 后台检测

后台脚本必须在启动后按固定周期检测代理服务器连通性。

后台脚本至少需要包含以下可配置项：

- 代理检测主机名或 IP
- 代理检测端口
- 代理绕过列表，默认排除本机和常见局域网地址
- 轮询间隔秒数

后台检测逻辑如下：

1. 读取当前机器级代理是否启用。
2. 探测代理地址和端口是否可达。
3. 当系统当前为“代理启用”且探测失败时，自动关闭机器级代理，切换到直连。
4. 当系统当前为“代理禁用”且探测成功时，自动重新启用机器级代理。
5. 其余情况保持当前状态，不重复切换。
6. 后台脚本必须同时同步 WinINet 机器级代理和 WinHTTP 机器级代理，确保未登录阶段的系统网络组件也能按相同策略工作。

### 3.2 开机运行

后台检测脚本必须支持在系统启动时自动运行，并且不依赖用户登录。

后台脚本必须以管理员权限运行，推荐直接使用 `SYSTEM` 账户。

预期部署方式为 Windows 计划任务：

- 触发器：系统启动时
- 运行账户：`SYSTEM`
- 允许运行于未登录状态

### 3.3 托盘显示

托盘脚本必须在用户登录后启动，并显示系统托盘图标。

托盘显示规则如下：

1. 图标主状态表示“系统当前机器级代理是否启用”。
2. 启用代理时显示“代理启用”。
3. 禁用代理时显示“直连模式”。
4. 托盘文本可补充最近一次检测结果，例如“最近检测成功”或“最近检测失败”。

预期部署方式为 Windows 计划任务：

- 触发器：用户登录时
- 运行账户：当前登录用户

### 3.4 退出联动

托盘必须提供“退出”菜单。

当用户点击退出时：

1. 托盘脚本写入停止信号文件。
2. 托盘脚本立即退出自身。
3. 后台脚本在下一次轮询时读取停止信号后，先关闭机器级代理，切换到直连。
4. 后台脚本随后退出。
5. 后台脚本退出时清理停止信号文件。

## 4. 共享状态

后台脚本和托盘脚本通过公共共享文件交换状态。

共享目录固定为：

- `C:\ProgramData\AutoProxyCheck\`

共享文件包括：

- `stop_proxy_checker.flag`：停止信号文件
- `proxy_status.json`：后台状态文件

`proxy_status.json` 至少需要记录：

- 状态更新时间
- 后台是否运行
- 当前系统代理是否启用
- 最近一次探测是否成功
- 最近一次状态说明

托盘脚本应优先读取 `proxy_status.json` 展示状态；如果状态文件不存在，则回退为直接读取系统代理开关状态。

## 5. 路径与部署约束

脚本执行文件可以移动目录，但共享状态文件固定使用公共目录：

- `C:\ProgramData\AutoProxyCheck\stop_proxy_checker.flag`
- `C:\ProgramData\AutoProxyCheck\proxy_status.json`

计划任务中的脚本执行路径仍然是部署配置的一部分：

- 当脚本被移动到新目录后，计划任务中的 `-File` 路径必须同步更新。
- 共享目录不随脚本目录变化。

## 6. 验收标准

### 场景 1：代理可达

- 后台启动后保持机器级代理启用。
- 状态文件记录最近探测成功。

### 场景 2：代理失效

- 后台在一个轮询周期内自动关闭机器级代理。
- 系统切换到直连模式。
- 状态文件记录最近探测失败。

### 场景 3：代理恢复

- 后台在一个轮询周期内自动重新启用机器级代理。
- 状态文件记录最近探测成功。

### 场景 4：用户登录

- 托盘图标正常显示。
- 托盘图标状态与系统当前机器级代理启用状态一致。

### 场景 5：退出托盘

- 托盘点击退出后立即消失。
- 后台在退出前自动关闭机器级代理，切换到直连。
- 后台在下一轮检测内退出。
- 停止信号文件被后台清理。
